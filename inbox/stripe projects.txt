Stripe projects

Stripe Projects (launched in developer preview ~March 26, 2026) is a new Stripe CLI plugin that lets developers (and AI coding agents) provision, manage, and pay for a full production-grade app stack directly from the terminal—no dashboards, no manual sign-ups, no copy-pasting keys. ￼
It creates a “project” (tied to one codebase) that links your Stripe account to third-party providers (Vercel, Supabase, Neon, PlanetScale, Clerk, PostHog, Runloop, Railway, Turso, Chroma, etc.). You run commands like:
•  stripe projects init my-app
•  stripe projects add vercel/project or supabase/database
•  stripe projects upgrade <provider>
•  stripe projects env --pull (syncs secrets to .env)
•  stripe projects llm-context (generates a single file with your entire stack context for agents)
Credentials are vaulted securely, upgrades/downgrades are billed via your Stripe account (using Shared Payment Tokens in preview regions), and everything works non-interactively for agents. ￼
Recommended architecture to make your products “Stripe Projects–native” and future-proof
If you ship products that include an engine, skills, tools, and wrappers (especially AI/agentic dev tools, coding agents, or infrastructure platforms), here’s the minimal, high-leverage architecture to add now so your users (and agents) can provision/manage your service seamlessly inside Stripe Projects.

If you want your own service to appear in the stripe projects catalog (i.e., become a first-class provider)
Contact provider-request@stripe.com with:
•  Provider name + primary business + engineering contacts
•  Category (hosting, database, auth, analytics, vector DB, sandbox/AI, etc.)
•  Short description + GitHub handles
•  Confirmation you can start building immediately
Stripe uses a co-designed integration protocol for provisioning, plan selection, credential handoff, and billing. Launch partners (Vercel, Supabase, PlanetScale, Runloop, etc.) already did this and are live. ￼
Quick-start checklist for your products right now (developer preview)
1.  Install the plugin yourself: stripe plugin install projects + request access at projects.dev.
2.  Add the four components above to your engine/skills/tools/wrappers (start with the CLI wrapper + LLM-context support — it’s only ~1–2 days of work).
3.  Document the exact commands your users/agents should run (or expose them via your own high-level SDK).
4.  Test end-to-end with a coding agent: “Spin up my service + a Postgres DB + auth in one project.”
5.  Watch for GA (expected soon after Sessions 2026 in late April).
Bottom line: Stripe Projects is the new “zero-friction infra layer” for the terminal/agent era. By baking CLI-first provisioning, agent-ready credentials, and llm-context into your engine/skills/tools/wrappers today, your products become the default choice whenever someone (or their agent) starts a new app. You’ll be ahead of everyone still forcing users into dashboards and manual API keys.
