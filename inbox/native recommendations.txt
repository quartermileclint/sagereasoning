Stripe Projects–Native Architecture Recommendations
Component
What to build / change
Why it matters for compatibility
Priority
Engine (core orchestration / runtime)
• Add support for running Stripe CLI commands (or call the underlying plugin APIs when public).
• Store and version .projects/ directory state alongside your own project config.
• Auto-merge/sync .env vars and handle vault rotation events.
• Support CI/CD flags (--json, --no-interactive, --auto-confirm, --quiet).
Projects lives in the local dev workspace. Your engine must treat it as first-class infrastructure (like Docker Compose or Terraform but Stripe-managed).
High
Skills (agent capabilities)
• New agent skill: “Provision / upgrade / rotate / remove service via Stripe Projects”.
• Skill that calls stripe projects catalog, add, upgrade, rotate, status, llm-context.
• Skill that interprets stripe projects llm-context output as system prompt context.
Stripe explicitly built this for coding agents. Agents that can fluently speak “Stripe Projects” will be dramatically more reliable at spinning up real infra.
Critical
Tools (function-calling / tool-use layer)
• Official wrapper / tool definition for the Stripe Projects CLI (one tool per major command, or a unified stripe_projects tool with sub-actions).
• JSON-mode output parsing + error recovery (e.g., “payment method missing → run stripe projects billing add”).
• Secure credential injection (never expose raw vault tokens).
Turns the CLI into a proper agent tool. Agents can now discover, provision, and pay for your service in one turn.
High
Wrappers / SDKs
• Language-specific SDKs (Node, Python, Go, etc.) that wrap the common CLI flows and expose typed methods (projects.add("your-provider/your-service")).
• Auto-generate llm-context snippets that describe your service’s capabilities, pricing tiers, and credential shape.
• Optional: provider-side integration (see below).
Makes it dead-simple for developers/agents using your SDK to stay inside the Stripe Projects flow instead of leaving the terminal.
Medium-High
