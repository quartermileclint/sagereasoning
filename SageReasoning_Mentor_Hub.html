Support Agent Manual

1. The Ring Architecture

Every action passes through the Sage Mentor ring: BEFORE (pre-check) then DRAFT then AFTER (post-evaluation). You get a vetted draft to review and approve.

The ring now also wraps Cowork sessions via the Session Bridge (Observer, Consultant, or Companion mode).

2. Governance Rules (R1-R14)

RuleSummary
R1No therapy — philosophical practice only
R2No employment evaluation
R3Disclaimer on all evaluative output
R4IP protection — results, not frameworks
R5Cost guardrail — paid covers 2x LLM costs
R6Methodology — qualitative, no numeric scoring
R7Source fidelity — trace to primary citations
R8Glossary enforcement — 3-tier terminology
R9No outcome promises
R10-R14Marketplace, wrappers, skills, embedding, compliance pipeline
Enforced automatically by the ring. Governance flags appear in ticket frontmatter.

3. Ticket Lifecycle

Open → Drafting → BEFORE check → Review → Resolved (or Escalated). Ticket files are markdown in support/inbox/ with YAML frontmatter.

Escalated tickets are assigned to you. The agent does not draft for governance-flagged items.

Promotion is automatic and deterministic. Demotion is protective, not punitive.

5. Workflow Playbooks

Six pre-built playbooks in workflows/: new-customer, enterprise, api-support, stripe-payment-failed, governance-flagged, weekly-pattern-summary. Create custom playbooks with trigger conditions and escalation rules.

6. Journal Upload & Mentor Profile

Three-stage pipeline: Chunking (by phase) → Extraction (AI analysis) → Aggregation (unified MentorProfile). Profile includes: passion map, causal tendencies, value hierarchy, oikeiosis map, virtue profile, key passages.

Profile is diagnostic, not punitive. Strong passions are mapped to focus guidance.

7. Session Bridge (New)

Connects Cowork sessions to the ring. Three modes:

  • Observer: Batch evaluation after session ends. Near-zero cost.
  • Consultant: On-demand ring check mid-session.
  • Companion: Live parallel evaluation with scrolling opinion feed.

Companion mode auto-activates for sessions matching strategic keywords. All classified exchanges persist to Supabase and the semantic memory layer.

Knowledge Context Summary auto-updates at end of sessions with strategic decisions.
