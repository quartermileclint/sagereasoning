## Rule-by-Rule Assessment

### R1 — No Therapeutic Implication
**Status: COMPLIANT**

- Document scorer evaluates "ideas, arguments, tone, and ethical posture" — not the author's psychology.
- System prompts explicitly state: "You assess the CONTENT of the document... NOT the author personally."
- Social media filter identifies passions in the content, not in the user psychologically. Labels: "Your Motivating Passions" (referring to what drove the writing), "What This Triggers in Readers" (content analysis, not psychological profiling).
- Policy review flags clauses by "passion exploited" — analysing document rhetoric, not assessing individuals.
- No language suggesting therapy, counseling, diagnosis, or treatment of persons.

### R3 — Disclaimer on Evaluative Output
**Status: COMPLIANT**

- `DOCUMENT_EVALUATIVE_DISCLAIMER` constant: "This is a philosophical framework and does not consider legal, medical, financial, or personal obligations."
- All three page UIs display this disclaimer below results.
- All three V3 prompts require `disclaimer` field in JSON output.
- Policy page displays disclaimer both at top (in header) and below results.

### R4 — Intellectual Property Protection
**Status: COMPLIANT**

- `V3_DOCUMENT_SCORING_PROMPT`, `V3_POLICY_SCORING_PROMPT`, and `V3_SOCIAL_MEDIA_PROMPT` are in document-scorer.ts, which is imported only by server-side API routes.
- Comment on relevant sections: "R4: This prompt is server-side only."
- API responses return evaluation results (proximity, passions, false judgements) but not the prompt template, evaluation sequence design, or diagnostic methodology.
- Badge displays proximity level only — no evaluation logic exposed.

### R6b — No Independent Virtue Weights
**Status: COMPLIANT**

- V3 prompts explicitly state: "do NOT score them independently" and "These are observations feeding a single unified assessment, NOT independent scores."
- No per-virtue numeric scores in V3 types or UI.
- `V3DocumentEvaluation` has `virtue_domains_engaged` (which domains were relevant) not individual virtue scores.
- Document page shows virtue domains as tags, not as scored bars.

### R6c — Qualitative Proximity, Not Numeric
**Status: COMPLIANT**

- All three tools output `katorthoma_proximity` as one of 5 qualitative levels.
- No 0-100 types in any V3 interface.
- Badge displays English proximity label ("Deliberate"), not a number.
- Proximity scale rendered as labeled segments, not numeric bar.

### R8c — User-Facing English Only
**Status: COMPLIANT**

- Document page: "Right Action Proximity," "Control Filter," "Appropriate Action," "Passions Identified," "Path Forward." All English.
- Policy page: "Deliberation Framework," "Who This Policy Affects," "Flagged Clauses." All English.
- Social page: "Social Media Filter," "Run Passion Diagnostic," "Your Motivating Passions," "What This Triggers in Readers," "False Judgements Embedded," "Correct Judgements." All English.
- Root passion display uses `ROOT_PASSION_ENGLISH` lookup: Craving, Irrational Pleasure, Fear, Distress.
- Proximity display uses `PROXIMITY_ENGLISH` lookup: Reflexive, Habitual, Deliberate, Principled, Sage-Like.
- No Greek or Latin terms appear on any screen.

### R9 — No Outcome Promises
**Status: COMPLIANT**

- No tool promises that acting on its analysis will produce specific outcomes.
- Social filter removed V1's "Good to publish" recommendation — V3 provides proximity level and corrections without prescribing whether to publish.
- Policy review does not frame flagged clauses as legal advice — the disclaimer explicitly excludes legal obligations.
- Improvement path in document scorer says "which false judgement to correct and which passion to address" — framed as philosophical reflection, not prescription.
