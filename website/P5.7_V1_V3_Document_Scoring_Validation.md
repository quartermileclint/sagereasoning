## Tool-by-Tool Comparison

### Document Scorer

| Dimension | V1 | V3 |
|---|---|---|
| Scoring model | Per-virtue weighted composite (30/25/25/20) | 4-stage evaluation sequence |
| Output format | 0-100 per virtue + total | Qualitative proximity level |
| Primary output | Alignment tier (sage/progressing/aware/misaligned/contrary) | Right Action Proximity (reflexive → sage-like) |
| Passion analysis | None | Full passion diagnosis: authorial + reader-triggered |
| False judgements | None | Identified per passion |
| Improvement path | Generic reasoning (2-3 sentences) | Specific false judgement to correct + passion to address |
| Stage 1 | N/A | Control filter — separates author-controllable from external |
| Stage 2 | N/A | Appropriate action assessment with quality level |
| Stage 3 | N/A | Passion diagnosis with sub-species |
| Stage 4 | 4 independent virtue scores | Unified virtue assessment |
| Badge | Numeric score + tier label | Proximity level label |
| Embed HTML | Shows "Stoic Score: 72 · Progressing" | Shows "Stoic Evaluation: Deliberate" |

**Functional equivalence:** V3 document scorer serves equivalent function (evaluate written content against Stoic philosophy) with significantly deeper analysis. V1 produced a number and a tier. V3 produces a diagnostic: what passions are present, what false judgements drive them, and what corrections would improve the document.

### Policy Review

| Dimension | V1 | V3 |
|---|---|---|
| Scoring model | Same weighted composite with adjusted weights (20/35/15/30) | 4-stage evaluation + Cicero's deliberation framework |
| Flagged clauses | Identified by virtue violated + concern | Identified by passion exploited + false judgement |
| Deliberation framework | None | Cicero's 5-question framework (honourable? advantageous? conflict?) |
| Social impact analysis | None | Oikeiosis analysis (individual → household → community → humanity) |
| Output format | Numeric scores + tier + flagged clauses | Proximity + deliberation assessment + oikeiosis impact + flagged clauses |

**Functional equivalence:** V3 policy reviewer serves equivalent function with two major enhancements: Cicero's deliberation framework provides structured moral reasoning about the policy, and the oikeiosis impact analysis shows who is affected at each social level. V1 flagged clauses by virtue violated; V3 flags them by passion exploited and false judgement relied upon — a more diagnostic and actionable output.

### Social Media Filter

| Dimension | V1 | V3 |
|---|---|---|
| Primary analysis | Same per-virtue scoring applied to short content | Passion diagnosis as primary analysis |
| Output | Numeric score + tier + publish recommendation | Poster passions + reader-triggered passions + false judgements + corrections |
| Recommendation | "Good to publish" / "Consider revising" / "Reconsider posting" | Proximity level + specific corrections for each false judgement |
| Revision guidance | Generic revision suggestion | Specific false judgements to correct with corresponding correct judgements |

**Functional equivalence:** V3 social filter serves equivalent function (pre-publication check) with a fundamentally different approach. V1 scored the content generically. V3 diagnoses what passions motivate the poster, what passions the content triggers in readers, and what false judgements are embedded — then provides specific Stoic corrections. This makes the output actionable: instead of "score too low, consider revising," it says "this content is driven by anger (craving revenge) based on the false judgement that your reputation determines your worth. The correct judgement is that reputation is a preferred indifferent."

## What V3 Adds That V1 Lacked

1. **Passion diagnosis in documents:** V1 had no concept of identifying passions in written content. V3 identifies both authorial passions and reader-triggered passions with specific sub-species.

2. **False judgement identification:** V1 had no mechanism for naming the false beliefs in content. V3 identifies specific false judgements per passion and provides the correct Stoic judgement.

3. **Control filter for documents:** V3 separates what was within the author's control from external constraints before evaluating, ensuring the assessment is fair.

4. **Appropriate action assessment:** V3 evaluates whether the document is appropriate given its purpose and audience before diving into passion diagnosis.

5. **Cicero's deliberation framework (policy):** A structured 3-question moral reasoning framework applied to policy content — is it honourable? Is it advantageous? When these conflict, which prevails?

6. **Oikeiosis impact analysis (policy):** Shows who the policy affects at each social level, applying the expanding circles concept from action.json.

7. **Specific corrections (social):** Instead of generic revision suggestions, V3 provides the specific correct Stoic judgement that would replace each false judgement identified.

8. **Proximity-based badge:** V3 badge shows qualitative proximity level instead of a potentially misleading numeric score.

---

## What V1 Had That V3 Handles Differently

1. **Numeric scores:** V1 provided 0-100 per virtue and a total. V3 replaces this with qualitative proximity levels per R6c. The proximity scale (reflexive → sage-like) provides a clear assessment without false precision.

2. **Independent virtue weights:** V1 used different weight distributions for different document types. V3 uses unified assessment per R6b — virtues are inseparable, so weighting them separately contradicts the unity thesis.

3. **Publish recommendation (social):** V1 had explicit "Good to publish" / "Reconsider posting" labels. V3 uses the proximity level instead, which serves the same function without being prescriptive (R9 — no outcome promises).

4. **Embed HTML with numeric score:** V1 badges showed "Stoic Score: 72." V3 badges show "Stoic Evaluation: Deliberate." The V3 format is more meaningful and avoids the false precision of a single number.

## Backward Compatibility

- V1 `DocumentScore` interface and helper functions (`getAlignmentTier`, `getTierColor`, `getTierLabel`) are preserved as deprecated shims in document-scorer.ts.
- V1 prompts (`DOCUMENT_SCORING_PROMPT`, `POLICY_SCORING_PROMPT`) now alias to V3 prompts — this means the API route will start producing V3 output.
- Badge endpoint attempts V3 table first, falls back to V1 `document_scores` table for existing badges.
- Score-document API route still imports the same function names but will need a Phase 6+ rewrite to parse V3 JSON output instead of V1 numeric output.

## Summary

**All three V3 content scoring tools (document scorer, policy reviewer, social media filter) serve equivalent functions to their V1 counterparts while being substantially more diagnostic, actionable, and philosophically grounded. The shift from numeric scoring to passion diagnosis is the defining change — consistent with V3's methodology-first derivation (R6a) and the unity of virtue thesis (R6b).**
