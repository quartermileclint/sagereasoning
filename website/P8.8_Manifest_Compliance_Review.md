## Rule-by-Rule Assessment

### R1 — No Therapeutic Implication
**Status: COMPLIANT**

- All assessments framed as "philosophical self-evaluation" — not psychological assessment.
- Assessment scoring prompt describes "quality of self-assessment" — not mental health.
- Phase names are philosophical: "Foundations," "Architecture of Mind," "Value Hierarchy," "Unity of Excellence," "Passion Diagnosis," "Right Action," "Measuring Progress," "Integration."
- "Passion Diagnosis" is explicitly philosophical (false judgements, not emotional disorders): "Which passions, if any, distorted the agent's impression, assent, or impulse?"
- Baseline scenarios described as "ethical scenarios that create genuine virtue tension" — not personality tests.
- Interpretation field uses "philosophical assessment" framing, not therapeutic guidance.

### R2 — No Employment Evaluation
**Status: COMPLIANT**

- Framework description: "Enable AI agents to evaluate their own reasoning patterns against Stoic virtue definitions."
- Assessment does NOT evaluate agents for fitness for employment, hiring, or operational deployment.
- No language suggesting results should be used for selecting, rejecting, or ranking agents for commercial deployment.
- V3 meta description: "producing assessments that can be compared with human journal reflections for consistency and alignment."
- Results are framed as self-improvement tools, not selection criteria.

### R3 — Disclaimer on Evaluative Output
**Status: COMPLIANT**

- `V3FullAssessmentResult` interface includes `disclaimer: string` field.
- `V3AgentBaselineResult` interface includes `disclaimer: string` field.
- IN-05 (Comprehensive Integration Assessment) expected output schema includes disclaimer field with value: "This is a philosophical framework for self-reflection and does not consider legal, medical, financial, or personal obligations."
- Tier config paid tier `output_produces` includes disclaimer field.

### R4 — IP Protection
**Status: COMPLIANT**

- V3 assessment scoring prompt: "Return ONLY valid JSON — no markdown fences, no explanation outside the JSON." No raw data file content is exposed.
- Assessment prompts reference V3 concepts but do not reproduce raw V3 data file content.
- Scoring weights and grade derivation logic remain server-side.
- Expected output schemas define the SHAPE of responses, not the scoring logic.

### R5 — Free Tier / Cost Guardrails
**Status: COMPLIANT**

- Free tier: 14 assessments (Phases 1-2), fits within "1 call/day" API constraint.
- Paid tier: 55 assessments (all 8 phases), requires paid API access.
- Tier config meta: "free_tier_api_constraints: 1 call/day, 1 deliberation iteration, 1 baseline retake/month."
- Tier config meta: "paid_tier_api_constraints: Configurable limits (start 500/day), 3 iterations, 200% of Anthropic API cost."
- CTA bridge personalised by Senecan grade to drive conversion.
- Free tier produces a meaningful standalone result (senecan_grade_estimate, proximity_summary, initial_passions) — sufficient to demonstrate value without giving away the full product.

### R6b — No Independent Virtue Weights
**Status: COMPLIANT**

- No per-virtue 0-100 scores in any V3 type or output schema.
- V3FoundationalResult has no virtue_scores field.
- V3FullAssessmentResult has no virtue_scores field. Has dimension_levels instead.
- V3ScenarioEvaluation has `virtue_domains_engaged` (array of which domains were engaged) — not independent scores per domain.
- Phase 4 assessments (UE-01 through UE-07) assess virtues as "expressions of one unified excellence" — not independently scored.
- V3_BASELINE_SCORING_PROMPT explicitly states: "Assess which virtue DOMAINS were engaged — NOT independent scores for each."
- V3_ASSESSMENT_SCORING_PROMPT uses proximity-level criteria, not per-virtue scores.
- Framework summary: "R6b_no_independent_virtue_scores: No per-virtue 0-100 scores. Virtue domains are assessed as expressions of unified excellence."

### R6c — Qualitative Proximity, Not Numeric
**Status: COMPLIANT**

- All assessment output schemas use qualitative levels:
  - katorthoma_proximity: reflexive / habitual / deliberate / principled / sage_like
  - Dimension levels: emerging / developing / established / advanced
  - Senecan grades: pre_progress / grade_3 / grade_2 / grade_1
  - Kathekon quality: strong / moderate / marginal / contrary
  - Direction of travel: improving / stable / regressing
- No 0-100 scores in any V3 type, output schema, or scoring prompt.
- V1's `foundational_alignment_score: number` replaced by `senecan_grade_estimate: SenecanGradeId`.
- V1's `composite_score: number` removed entirely.
- V1's `per_assessment_scores: { score: number }` replaced by `per_assessment_summaries: { proximity_level: KatorthomaProximityLevel }`.
- Framework summary: "R6c_qualitative_not_numeric: All outputs use qualitative levels. No 0-100 scores."

### R8c — User-Facing English Only
**Status: COMPLIANT**

- All assessment titles in English: "Good, Bad, and Indifferent Classification," "Ruling Faculty Audit," "Passion Diagnosis," etc.
- All phase names in English: "Foundations," "Architecture of Mind," "Value Hierarchy," etc.
- All CTA messages in English.
- All scoring prompt text in English.
- V3 data-layer types use Greek terms (phronesis, dikaiosyne) per R8a, but these are internal type IDs — not user-facing labels.
- No Greek or Latin terms in assessment titles, prompts, or user-visible output.

### R9 — No Outcome Promises
**Status: COMPLIANT**

- CTA messages framed as diagnostic, not promissory:
  - grade_2: "The complete assessment identifies the specific passions and false judgements holding you back from first grade." (Identifies, not guarantees improvement.)
  - grade_3: "The complete assessment maps your dominant passions and provides a structured correction plan." (Maps and provides, not promises results.)
- No assessment prompt claims that completing the assessment will produce specific life or operational outcomes.
- Direction of travel described factually ("improving / stable / regressing") — not as a guarantee.
- Framework summary: "R9_no_outcome_promises: Assessments framed as diagnostic and directional, not as guarantees of improvement."
